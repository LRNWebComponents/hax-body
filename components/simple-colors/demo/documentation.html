simple-colors documentation

To use simple-colors in a component:
  1. bower install --save LRNWebComponents/simple-colors
  2. Include <link rel="import" href="path/to/simple-colors/simple-colors.html">
  3. To allow an element to inherit the variables, add the behavior: simpleColorsBehaviors,
  4. When the element ready, request the color variables: Polymer.SimpleColorsUtility.requestAvailability();
  5. Add custom styles to your element using the simple colors variables. For example:
    <style is="custom-style" import="simple-colors">
        :host {
            color: --simple-colors-accent-foreground1;
            --paper-listbox-background-color: --simple-colors-accent-background2;
            border-color: 1px solid var(--my-component-border-color, --simple-colors-red-background3);
        }
    </style>
  6. …Or use the simple-colors text, background and border color classes. For example:
    class="simple-colors-light-blue-foreground1-text simple-colors-light-blue-background5-border simple-colors-lime-background1"

The naming convention for simple-colors CSS variables is: --simple-colors-(theme name)-(color)-foreground or background with level. For example, --simple-colors-light-theme-red-background1 would be a highest-contrast background color for UI element with a lighter theme. (See color reference below.)

There are also CSS classes. The convention for simple-colors CSS classes is:

  • Background color: simple-colors-(color)-foreground or background with level
  • Text color: simple-colors-(color)-foreground or background with level-text
  • Border color: simple-colors-(color)-foreground or background with level-border
Note that the classes do not allow you to specify a theme. For example, <div class="simple-colors-foreground1-text simple-colors-red-background3-border simple-colors-red-background1"/> would be a highest-contrast text color, highest-contrast background color, and a mid-condrast border-color. (See color reference below.)

theme name

The theme name, is optional. It can be light-theme or dark-theme.

The --simple-colors-light-theme prefixed colors are based on a light foreground with a dark background. The --simple-colors-dark-theme prefixed colors are based on a dark foreground with a light background. These colors will not change based on an element (or parent's) dark.

If the theme name is omitted, the theme will be applied based on whether or not the element or its ancestor has a dark attribute. This allows you to specify only one set of CSS color variables and allow a single attribute to switch between light and dark. In other words, you can change --simple-colors-foreground4, --simple-colors-red-foreground1, --simple-colors-background4, and --simple-colors-red-foreground1 just by changing an element's dark attribute.

color

The color, is optional. If omitted, the color will be in greyscale. Simple colors using the naming and similar shares of Materialize. However, colors have been adapted to support WCAG 2.0 AA contrast ratios. The colors availble are: red, pink, purple, deep-purple, indigo, blue, light-blue, cyan, teal, green, light-green, lime, yellow, amber, orange, deep-orange, brown, and blue-grey.

There is also an accent color that changes based on an the accent-color attribute of an element or its ancestor. For example, if accent-color="indigo", then --simple-colors-accent-foreground2 will be set to --simple-colors-indigo-foreground2. If your accent color is indigo, but you want to use red for a delete button, you can either, set the button to accent-color="red" to override the parent's color, or you can use CSS to change its color to --simple-colors-red-background1.

foreground or background with level

For each theme there are 5 foreground and 5 background greyscale values, as well as 3 foreground and 3 background color values. The lower the number, the higher the contrast.

Most often you will want to use the highest-contrast foreground with the highest-contrast background to ensure readability. For short sections of text, such as callouts or UI buttons, you may reduce the contrast.

Small Text

Small text requires the highest contrast ratio for WCAG 2.0 AA compliance. Small text is any content that is NOT:

  • bold text that is 14 point or higher,
  • text that is 18 point or higher,
  • a decorative elements that has no semantic meaning, such as a border, or
  • a disabled UI element

Large Text

WCAG 2.0 AA requires higher contrast for small text than for large text and for UI elements that have no text. Large text is defined as:

  • bold text that is 14 point or higher, or
  • text that is 18 point or higher

Other UI Elements

WCAG 2.0 AA allows the following exceptions for contrast:

  • decorative elements that have no semantic meaning, such as borders
  • disabled UI elements
